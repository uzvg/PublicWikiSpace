## 一、什么是 dataclass？

`dataclass` 是 Python 3.7+ 提供的一个工具，用来**快速定义“以数据为核心”的类**。

👉 本质：**自动帮你生成样板代码（boilerplate）**

## 二、dataclass vs 普通类

### 1️⃣ 打印行为区别

```python
class A:
    def __init__(self, x):
        self.x = x

print(A(1))
# <__main__.A object at 0x...>
```

```python
from dataclasses import dataclass

@dataclass
class B:
    x: int

print(B(1))
# B(x=1)
```

✅ dataclass 自动生成 `__repr__`，更适合调试

## 三、自动生成的方法

默认情况下，`@dataclass` 会生成：

| 方法         | 作用       |
| ---------- | -------- |
| `__init__` | 自动初始化字段  |
| `__repr__` | 打印对象信息   |
| `__eq__`   | 比较对象是否相等 |

### 🔍 `__repr__` vs `__str__`

| 方法         | 用途        |
| ---------- | --------- |
| `__repr__` | 面向开发者（调试） |
| `__str__`  | 面向用户（展示）  |

规则：

* `print(obj)` 👉 优先调用 `__str__`
* 没有 `__str__` 👉 fallback 到 `__repr__`

👉 dataclass 默认只生成 `__repr__`

## 四、@dataclass 参数详解（重点补全）

```python
@dataclass(
    init=True,
    repr=True,
    eq=True,
    order=False,
    unsafe_hash=False,
    frozen=False,
    match_args=True,
    kw_only=False,
    slots=False,
    weakref_slot=False
)
```

### 📊 参数说明表

| 参数             | 作用              | 默认值   | 关键理解         |
| -------------- | --------------- | ----- | ------------ |
| `init`         | 是否生成 `__init__` | True  | 很少关闭         |
| `repr`         | 是否生成 `__repr__` | True  | 调试用          |
| `eq`           | 是否生成 `__eq__`   | True  | 比较对象         |
| `order`        | 生成 `< <= > >=`  | False | 依赖 eq        |
| `unsafe_hash`  | 强制生成 hash       | False | ⚠️ 慎用        |
| `frozen`       | 是否不可变           | False | 类似“只读对象”     |
| `match_args`   | 支持模式匹配          | True  | Python 3.10+ |
| `kw_only`      | 字段必须用关键字传参      | False | 提高可读性        |
| `slots`        | 使用 `__slots__`  | False | 节省内存         |
| `weakref_slot` | 支持弱引用           | False | 高级特性         |

## 五、weakref 是什么？（你问的重点）

### 🧩 概念

`weakref`（弱引用）允许你：

👉 **引用对象，但不增加其引用计数**

### 🔥 为什么重要？

Python 的垃圾回收机制：

👉 对象引用计数为 0 才会被释放

普通引用：

```python
a = obj
```

👉 会阻止对象释放

弱引用：

```python
import weakref
weakref.ref(obj)
```

👉 **不会阻止对象被回收**

### 🧠 `weakref_slot=True` 的作用

当你开启：

```python
@dataclass(slots=True, weakref_slot=True)
```

👉 允许这个对象被弱引用

### ⚠️ 使用场景（不常见但关键）

* 缓存系统（避免内存泄漏）
* 观察者模式
* GUI 对象管理

👉 简单说：

> 不想“控制对象生命周期”，但想“临时引用它”

## 六、`field()` 函数

用于**精细控制字段行为**

### 🧪 示例

```python
from dataclasses import field

scores: list[int] = field(default_factory=list)
```

### ⚠️ 为什么不用 `scores = []`？

```python
scores = []
```

👉 所有实例共享同一个 list ❌

```python
default_factory=list
```

👉 每个实例独立创建 ✅

### 📊 常用参数

| 参数                | 作用               |
| ----------------- | ---------------- |
| `default`         | 默认值              |
| `default_factory` | 工厂函数（推荐）         |
| `init`            | 是否加入 `__init__`  |
| `repr`            | 是否出现在 `__repr__` |
| `compare`         | 是否参与比较           |
| `metadata`        | 自定义扩展信息          |

## 七、类变量（ClassVar）

```python
from typing import ClassVar

@dataclass
class A:
    x: int
    count: ClassVar[int] = 0
```

### 🔍 特点

* 不属于实例
* 不参与：

  * `__init__`
  * `__repr__`
  * `__eq__`

👉 本质就是普通类变量，但 dataclass **会忽略它**

## 八、`__post_init__()`（非常关键）

### 🧠 执行时机

```text
__init__() 执行完 → 自动调用 __post_init__()
```

### 🎯 常见使用场景

#### 1️⃣ 派生字段计算

```python
@dataclass
class A:
    x: int
    y: int
    sum: int = 0

    def __post_init__(self):
        self.sum = self.x + self.y
```

---

#### 2️⃣ 参数校验

```python
def __post_init__(self):
    if self.x < 0:
        raise ValueError("x must be >= 0")
```

---

#### 3️⃣ 类型转换

```python
def __post_init__(self):
    self.x = int(self.x)
```

---

👉 核心一句话：

> `__init__` 做“赋值”，`__post_init__` 做“逻辑”
